## Workflow

### One

# The Actual Workflow: Captured Intelligence

**The Complete Cycle:**

1. **Human defines task** (in text/voice → text)
   - Problem statement
   - Goals
   - Constraints

2. **Human selects context** (curates proven changelogs)
   - "Use these successful implementations as reference"
   - This stamping = validation propagation

3. **Conversation happens** (AI + Human ideation)
   - "What if we try X?"
   - "That won't work because Y"
   - "Better approach: Z"
   - All captured in conversation thread

4. **AI implements** (with human guidance)
   - Code changes made
   - Human provides corrections mid-stream
   - All reasoning captured in chat

5. **Human validates outcome** (creates changelog)
   - What was built
   - Why decisions were made
   - What problems it solved
   - What it enables next

**The Critical Point:**
Every step—the ideation, the rejected approaches, the "why not X", the corrections, the final reasoning—**exists as indexed text** in the conversation thread.

**What Gets Captured:**
- Initial problem framing
- Context selection (what you stamped as relevant)
- Design discussion (alternatives considered and rejected)
- Implementation decisions (why this approach over that)
- Validation outcome (changelog confirming success)

**The Compounding Effect:**
Next implementation can retrieve not just the final code, but the **entire decision context**—the conversation that led to it. That's the intellectual capital traditional development loses forever.

### Two

# Core Insight: Development as Genealogy

**The Fundamental Pattern:**
Every implementation creates three artifacts:
1. The code itself
2. A changelog capturing what was built and why
3. The context that was used to build it

**The Breakthrough:**
When a developer selects previous successful changelogs as context for a new task, they're creating a **genealogy of validated success**. This "stamping" act is the most powerful validation signal—human curation of what actually worked.

**The Simple Data Unit:**
Store only:
- Original implementation content
- One AI-generated summary (what it built upon, problems solved, what it enables)
- Simple relationships (built-upon, spawned, conflicts-with)

**Why This Works:**
Traditional documentation captures **what** was built. This captures **why it worked, what it built upon, and what it enables**—creating a searchable genealogy where each unit inherits intelligence from its ancestors.

**The Economic Insight:**
Changelogs are concise. This means:
- Cheap to embed continuously (milliseconds, not minutes)
- Can re-vectorize entire history in real-time
- Enables spontaneous pattern discovery without expensive infrastructure

**The Result:**
Implementation knowledge compounds through natural workflow rather than requiring separate documentation effort. The work *is* the documentation. The context *is* the institutional memory.

## Insight

**Agentic coding enables externalizing all intellectual work into persistent, indexed context.**

Traditional development captured implementation. AI agents can leverage the full decision graph—ideation, rejected approaches, constraints, reasoning—if it's preserved. Every conversation, design discussion, and micro-decision becomes compound intellectual capital.

Voice-to-text and conversational development lower the friction to capture thought as it happens. The opportunity: developers who externalize everything build exponentially more effective AI collaboration than those who code in silence.
