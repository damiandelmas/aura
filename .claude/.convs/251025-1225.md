  The Problem

  From that section_type: "Message 1: USER" string, we COULD parse:
  - Chunk type: "Message" vs "Code Patch"
  - Role: "USER" vs "ASSISTANT"
  - File path: "src/cli.py" (from patch headers)

  The Solution

  In imem/ingest.py, add parsing logic:

  def enrich_conversation_metadata(node):
      """Parse section_type string into structured metadata"""

      section_type = node.metadata.get('section_type', '')

      # Parse "Message 1: USER" or "Code Patch 1: src/cli.py"
      if section_type.startswith('Message'):
          # Extract role
          if 'USER' in section_type:
              return {'chunk_type': 'message', 'role': 'user'}
          elif 'ASSISTANT' in section_type:
              return {'chunk_type': 'message', 'role': 'assistant'}

      elif section_type.startswith('Code Patch'):
          # Extract file path
          match = re.match(r'Code Patch \d+: (.+)', section_type)
          file_path = match.group(1) if match else None
          return {'chunk_type': 'patch', 'file_path': file_path}

      return {}
