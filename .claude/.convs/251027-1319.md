  Composition Pattern (Claude Orchestrates)

  Slash Command: /explore-auth

  Prompt:
  "Search authentication from multiple angles:
  1. Decisions
  2. Failures
  3. Patterns
  Then find most authoritative via PageRank."

  Claude executes:
  r1 = imem develop search "auth" --decisions
  r2 = imem develop search "auth" --failures
  r3 = imem develop search "auth" --patterns

  graph = imem graph build <r1-ids> <r2-ids> <r3-ids>
  ranked = imem graph apply <graph> pagerank

  For top result:
  siblings = imem siblings <top-result-id>

  Response: Top decision + its constraints + failures

  CLI primitives. Claude composes. Slash command as reusable pattern.

  What Makes This "MCP-like"

  Traditional MCP:

  ├─ Claude calls via protocol
  └─ Structured input/output

  ├─ Prompt exposes intent
  ├─ Claude composes CLI tools
  └─ Bash input/output

  Difference: Granularity (protocol vs bash)

  Slash commands = MCP at bash level. Modular, iterable, observable.
