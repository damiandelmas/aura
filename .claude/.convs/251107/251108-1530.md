● PERFECT. Two distinct metadata systems:

  ---
  KNOWLEDGE METADATA (Visibility Network)

  Resources:
  - Documents (single files)
  - URLs (multiple pages/content)
  - Folders (messy collections: subfolders, scripts, etc)
  - Repos (git repositories)

  Metadata:

  Tier 1 (objective facts):
  source: https://graphiti.ai/docs
  type: documentation
  authority: 1
  tags: [graph-memory]

  Tier 2 (usage accumulation):
  {"session":"abc", "accessed":"2025-11-08", "context":"researching auth"}
  {"attention": 0.85, "serve_as": "reference"}

  Purpose: Make ALL knowledge visible to AI agents, whether indexed or not.

  ---
  CONTENT METADATA (Indexed Material)

  Resources:
  - Documents (markdown files - indexed)
  - Chunks (sections within documents)

  Metadata:

  Document level:
  type: "design"  # phase/category
  session_id: "abc123"

  Chunk level:
  {
    "inherits": "design",  // from document
    "section_types": ["Decision", "Implementation"],  // multi-label
    "embedding": [...]
  }

  Purpose: Semantic search + type filtering within indexed content.

  ---
  The Separation:

  KNOWLEDGE (broad)
  ├── Tier 1: Metadata wrappers for everything
  ├── Tier 2: Access logs for everything
  └── Content (narrow)
      ├── Documents: Indexed markdown
      └── Chunks: Vectorized + typed sections

  Key: Knowledge network > Content index. You can track/access repos without chunking them. Chunking is
  optional enrichment for semantic search.

  ---
  The files:

  access.jsonl      # KNOWLEDGE: All resource touches
  registry/         # KNOWLEDGE: Tier 1 wrappers
  chunks.jsonl      # CONTENT: Indexed + vectorized
  documents.jsonl   # CONTENT: Document metadata

  Two systems. Knowledge enables visibility. Content enables search.
