  The Workflow

  User types: /specify Build a photo gallery app
                ↓
  Slash command calls: create-new-feature.sh "Build a photo gallery app"
                ↓

                ↓

  Script Breakdown

  1. create-new-feature.sh (The Initializer)

  Called by: /specify command

  What it does:
  1. Finds repo root - Works with/without git
  2. Auto-numbers features - Scans specs/ for highest number (001, 002, 003...)
  3. Generates branch name - Converts description to kebab-case
    - "Build a photo gallery app" → 001-build-a-photo
  4. Creates git branch - git checkout -b 001-build-a-photo (if git available)
  5. Creates directory structure - specs/001-build-a-photo/
  6. Copies template - spec-template.md → spec.md
  7. Sets environment var - export SPECIFY_FEATURE="001-build-a-photo"
  8. Returns JSON - So agent knows where to write

  2. setup-plan.sh (The Planner)

  Called by: /plan command

  What it does:
  1. Sources common.sh - Gets all paths
  2. Validates feature branch - Ensures you're on a 00X-name branch
  3. Copies plan template - plan-template.md → plan.md
  4. Returns JSON:
  {
    "FEATURE_SPEC": "/path/to/spec.md",
    "IMPL_PLAN": "/path/to/plan.md",
    "SPECS_DIR": "/path/to/001-feature/",
    "BRANCH": "001-feature-name"
  }

  3. common.sh (The Utility Belt)

  Sourced by: All other scripts

  Functions:
  - get_repo_root() - Finds project root (git or .specify/ marker)
  - get_current_branch() - Gets branch name (or uses $SPECIFY_FEATURE env var)
  - get_feature_paths() - Returns all standardized paths:
  FEATURE_DIR='specs/001-feature/'
  FEATURE_SPEC='specs/001-feature/spec.md'
  IMPL_PLAN='specs/001-feature/plan.md'
  TASKS='specs/001-feature/tasks.md'
  RESEARCH='specs/001-feature/research.md'
  DATA_MODEL='specs/001-feature/data-model.md'
  CONTRACTS_DIR='specs/001-feature/contracts/'
  - check_feature_branch() - Validates 00X- naming
  - has_git() - Checks if git available

  4. check-prerequisites.sh (The Validator)

  Called by: /tasks, /implement commands

  What it does:
  - Validates required files exist
  - Returns list of available docs
  - Three modes:
    - --paths-only - Just output paths
    - --json - Return JSON
    - --require-tasks - Fail if tasks.md missing

  The Pattern

  All scripts follow:
  1. Accept --json flag
  2. Source common.sh for paths
  3. Do their specific job
  4. Return JSON or text output
  5. Agent parses JSON and continues workflow

  Non-Git Support

  Scripts work without git via:
  - $SPECIFY_FEATURE env var override
  - .specify/ directory marker fallback
  - Auto-detect latest feature number
  - Warning messages instead of hard failures

  - ✅ Path resolution
  - ✅ Validation

  Agent does the thinking - they handle:
  - ✅ Writing specs
  - ✅ Writing plans
  - ✅ Writing code

  Clean separation of concerns.
