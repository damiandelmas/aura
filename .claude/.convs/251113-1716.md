  ---
  Proposed Structure:

  .nexus/                          # MANAGE implementation
  ├── core/
  │   ├── access.jsonl            # All resource accesses (append-only)
  │   ├── registry.jsonl          # Tier 1: Objective facts about sources
  │   └── timeline.jsonl          # Git commit spine (per project)
  │
  ├── project/                     # Tier 2: Per-project qualification
  │   ├── attention.jsonl         # Usage accumulation scores
  │   ├── qualification.jsonl     # How sources are qualified for this project
  │   └── motivation.jsonl        # Why sources were accessed
  │
  └── .mind/                       # Intelligence layer
      ├── schema/
      │   └── evolved.jsonl       # Emerged canonical types
      ├── entities/
      │   └── mappings.jsonl      # Keyword variants → canonical
      ├── graphs/
      │   └── cache/              # Ephemeral graph results
      ├── validation/
      │   └── drift.jsonl         # Temporal cortex drift scores
      └── introspection/
          └── capabilities.json   # Exposed system schema

  ---
  Component Descriptions:

  access.jsonl
  - Logs every resource touch (read, webfetch, imem query, git operation)
  - Schema: {timestamp, session_id, source, method, context}
  - Foundation for all emergence

  registry.jsonl
  - Tier 1 entries: Objective facts about sources
  - Schema: {source, format, first_seen, retrieval_method}
  - Emerges from first access across all projects

  timeline.jsonl
  - Git commit history for this project
  - Schema: {hash, timestamp, message, files_changed[]}
  - Validation spine for Temporal Cortex

  qualification.jsonl
  - How this project qualifies each source
  - Schema: {source, serve_as, attention, last_accessed}
  - Determines authority at serve time

  evolved.jsonl
  - Canonical section types discovered by Schema Evolution
  - Schema: {canonical_type, variants[], observed_count}

  mappings.jsonl
  - Entity resolution: keyword variants → canonical
  - Schema: {canonical, variants[], domain}

  drift.jsonl
  - Temporal Cortex validation results
  - Schema: {source, commit_hash, divergence_radius, validated_at}

  capabilities.json
  - Introspection: what the system can do
  - Schema: {section_types[], entities[], queryable_fields[]}

  ---
  Dataflow:

  SOURCE ACCESSED
      ↓
  [Access Logger] → access.jsonl
      ↓
  First access? → registry.jsonl (Tier 1 emerges)
      ↓
  [Qualification] → qualification.jsonl (Tier 2 accumulates)
      ↓
  [MIND observes patterns]
      ↓
  evolved.jsonl + mappings.jsonl
      ↓
  [Query Time]
      ↓
  [Orchestrator] reads qualification → routes to retrieval method
      ↓
  [MIND enriches] with validation, graphs, entities
      ↓
  SERVE with authority
