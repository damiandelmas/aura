# aiUX: Memory for AI Comprehension

Before: I thought this was about serving information to users (developers) who query the system.

After: The USER is the AI agent. The UX is how we structure retrieval so AI agents don't repeat failed approaches. When Claude (or future agents) queries "mobile keyboard solutions," the system doesn't just return similar chunks—it returns genealogical context with explicit failure markers (❌ Don't Suggest), temporal evolution showing what came after failures, and visual indicators that route AI attention. Structure = comprehension. The breakthrough: treating presentation structure as the interface for AI understanding, not human reading. Decision genealogy over similarity ranking.
