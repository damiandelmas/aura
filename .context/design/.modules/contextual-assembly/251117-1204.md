**`temporal_position` Calculation** (compose.py:331-352):
```python
def _detect_temporal_position(result):
    if result['payload'].get('section_type') == 'Failures':
        return 'failed_branch'
    
    temporal = result.get('temporal', [])
    if not temporal:
        return 'current_thrust'  # ← DEFAULT WHEN NO DATA
    
    # Count later chunks
    later_chunks = [t for t in temporal if timestamp > result_timestamp]
    if len(later_chunks) > 2:
        return 'superseded'
    elif len(later_chunks) > 0:
        return 'evolved'
    else:
        return 'current_thrust'
```

**THE PROBLEM:**
- Line 341: `if not temporal: return 'current_thrust'`
- **Meaning:** "No temporal data found" → "This must be current"
- **Reality:** Could mean:
  - Actually is current (no superseding decisions)
  - OR temporal discovery returned empty (0.85 threshold too high)
  - OR chunk has no timestamp
  - OR this is the only chunk on this topic

**This is the LYING the user identified.**

## **WHAT DOESN'T WORK** ❌

### **1. Temporal Discovery Returns Empty**
- High similarity threshold (0.85) filters out most candidates
- OR data doesn't have enough temporal relationships yet
- Result: `temporal_position` defaults to "current_thrust" unreliably

### **2. "both" Direction Not Implemented**
- Code expects 'after' or 'before'
- Passing 'both' falls through (doesn't filter by direction)
- Works accidentally but not explicitly handled

### **3. No Validation Signals**
- `has_rationale`, `has_alternatives` are template compliance flags
- NOT validated by humans or code
- AI fills these during ingestion (self-reported)

### **4. No Graph Computation**
- `_apply_graph_operations()` exists (compose.py:365-398)
- Only does simple reference counting (siblings + genealogy length)
- No PageRank, betweenness, authority scoring
- Marked as placeholder (line 370 comment)

### **5. Template Makes Unvalidated Claims**
- Uses `temporal_position` without checking if temporal data exists
- Claims "CURRENT THRUST" based on absence, not evidence
- No distinction between:
  - "Confirmed current (checked, no supersession found)"
  - "Presumed current (no temporal data available)"
