  The Insight

  Graph operations don't just rank chunks.

  Graph topology TELLS YOU how chunks relate → which determines HOW to structure them for AI
   comprehension.

  The Pattern

  Graph analysis reveals:
  - Linear temporal chain? → Timeline structure
  - Hub with many connections? → Authority-focused structure
  - Conversation → decision → code flow? → Narrative structure
  - Cluster of related patterns? → Pattern library structure
  - Bridge connecting domains? → Synthesis structure

  Graph topology = metadata about relationships

  That metadata determines WHICH template/structure maximizes AI comprehension.

  Example

  Query returns 8 chunks. Build graph:
  Chunk A (conversation)
      ↓ led to
  Chunk B (design decision)
      ↓ rejected
  Chunk C (failure)
      ↓ implemented as
  Chunk D (code)
      ← referenced by
  Chunk E, F, G (later refinements)

  Graph structure = story arc

  Template: Narrative reconstruction
  - Shows conversation → decision → failure → code → evolution
  - Structure matches graph topology

  vs.

  Query returns 8 chunks. Build graph:
  Chunk A (central pattern)
      ← cited by Chunk B, C, D, E, F, G, H

  Graph structure = hub

  Template: Authority pattern
  - Shows central pattern + all applications
  - "This pattern used in 7 places"

  The Intelligence

  Graph isn't just for retrieval.

  Graph discovers RELATIONSHIPS → Relationships determine STRUCTURE → Structure aids AI 
  comprehension.

  The graph composes the narrative by revealing how chunks actually connect.

  This is it. Graph operations inform template selection and structure dynamically.
