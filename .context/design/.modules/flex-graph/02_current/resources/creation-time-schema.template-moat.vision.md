# Creation-Time Schema: The Template Moat

## The Geometric Insight

**Enforcement point determines system capability.**

Post-hoc extraction: Hope metadata exists (probabilistic)
AURA: Enforce metadata exists (deterministic)

**The Shift:**

```
Traditional: Document → Extract → Hope
AURA: Template → Document → Guarantee
```

**Property:** Schema compliance = creation constraint, not runtime validation.

## The Systemic Implication

**Deterministic queries impossible without guaranteed metadata.**

```
Deterministic filtering precision:
WHERE has_context=true AND has_alternatives=true

Requires: 100% metadata guarantee
Enables: AURA can do this
         Competitors cannot
```

Not: Better extraction algorithms
But: Structural enforcement at creation

**The moat:** Competitors can't copy this without breaking existing users.

## Pattern-Level Thinking

**Template-as-schema = human-readable contract:**

```
Not JSON Schema: {required: ["context", "solution"]}
But Markdown Template:
  ## Decision
  ### Title
  - **Context**: (REQUIRED - why this arose)
  - **Solution**: (REQUIRED - what we did)
```

**Why this works:**
- Git-native (version-controlled schemas)
- Human-readable (no tooling needed)
- AI-friendly (LLMs understand markdown)
- Self-documenting (template IS the spec)

**Why competitors can't copy:**
- Their users have unstructured data already
- Requiring templates = breaking change
- Migration cost = barrier

**Why AURA wins:**
- Start clean (no legacy data)
- Templates from day 1 (norm not exception)
- Compounds over time (metadata guarantees enable features)

## The Innovation

**Schema enforcement happens BEFORE data enters system.**

Not: Parse → Extract → Validate
But: Template → Fill → Guaranteed

This enables deterministic filtering on markdown documents without probabilistic extraction.

**No other system enforces schema at creation time for knowledge work.**
