# Flippable Chunks: Zero-Loss Memory Degradation

## The Insight

**Supersession ≠ deletion. Supersession = abstraction promotion.**

Traditional systems: Old knowledge deleted or marked obsolete → information loss.

AURA: Old implementation superseded → serve pattern abstraction → zero knowledge loss.

---

## The Geometry

```
Each decision = double-sided chip

Side A: Implementation (tech-specific)
Side B: Pattern (principle)

Default:    Show Side A (current implementation)
Superseded: Flip to Side B (timeless pattern)
On demand:  Flip back to Side A (archaeological precision)
```

**No deletion. Just perspective shift.**

---

## The Properties

**Zero-loss degradation:**
- Implementation expires → Pattern serves
- Full resolution preserved → Force flag retrieves
- Temporal archaeology available → `--full-resolution`

**O(1) flip, not O(n) reindex:**
- Supersession = metadata field
- Serving decision = query-time logic
- Both chunks indexed, serving determined dynamically

**Reversible:**
- Default: Pattern (abstraction)
- Override: Implementation (precision)
- User chooses resolution

---

## Why It Matters

**Traditional supersession:**
```
Decision v1 → Decision v2 replaces it
Result: v1 lost or marked obsolete
Problem: Can't study what actually happened
```

**Flippable supersession:**
```
Decision v1 → Decision v2 supersedes it
Result: v1.pattern serves by default, v1.impl available on demand
Benefit: Abstraction by default, precision when needed
```

**The shift:** From binary (keep/delete) to spectrum (resolution levels).

---

## The Vision

Every memory has multiple resolutions:
- Query default: Current or abstracted
- Query cross-project: Patterns only (anti-contamination)
- Query archaeological: Full implementation (debugging)

**Schrödinger's memory:** Resolution determined by observer, not storage.
