# Flippable Chunks: Zero-Loss Memory

## The Geometric Insight

**Supersession ≠ Deletion**

Traditional systems: Old knowledge dies when replaced
AURA: Old implementation dies → pattern lives

**The Board:**

```
Each decision = double-sided chip

Side A: Implementation (tech-specific, expires)
Side B: Pattern (language-agnostic, eternal)

Default: Show A
Superseded: Flip to B
On demand: Force A
```

**Properties:**
- Zero re-indexing (metadata flip, not data change)
- Zero loss (implementation preserved, pattern served)
- O(1) flip (metadata read, not graph traversal)

## The Systemic Implication

Knowledge doesn't degrade — it **abstracts**.

**Lifecycle:**
```
Create → Use → Supersede → Abstract → Reuse
```

Not:
```
Create → Use → Deprecate → Delete → Lost
```

**The difference:** Intellectual capital accumulates instead of churning.

## Pattern-Level Thinking

**Schrödinger's Memory:**
- Same chunk ID
- Different serving mode based on context
- Implementation OR pattern, runtime decision

**Authority via Abstraction:**
- Superseded implementation = validated pattern
- Pattern appearing in 5 projects = proven approach
- Cross-project PageRank on pattern layer = institutional wisdom

**Anti-Pattern Avoided:**
- Marking obsolete (loses value)
- Deleting old (loses history)
- Re-indexing (breaks references)
- Hiding deprecated (confuses context)

## The Innovation

**No other system serves abstraction on supersession.**

Most systems: Deprecate and hide
Some systems: Mark outdated, reduce ranking
AURA: Flip to pattern layer, preserve implementation

This enables knowledge to **evolve upward** (toward abstraction) instead of churning laterally (replacement cycles).
