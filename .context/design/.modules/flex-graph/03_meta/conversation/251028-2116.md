## 1. Full Genealogy (Most Common Use Case)

Shows primary decision + all related sections + conversation origin

```bash
imem compose '{"search": {"text": "trace chronicle refactor", "filters": {"phase": "develop"}, "limit": 1}, "discovery": {"siblings": true, "genealogy": true}, "output": {"template": "genealogy"}}'
```

**Returns:**
- Primary match with semantic score
- 15-20 sibling sections from same document
- Origin conversation (if indexed)
- Formatted markdown output

**Expected:** ~100ms

## Config Reference

### Search Stage
```json
"search": {
  "text": "query string",
  "filters": {"phase": "develop"},  // Optional: develop, design, document
  "limit": 5                          // Default: 10
}
```

**Multi-query:**
```json
"search": {
  "queries": [
    {"text": "query1", "filters": {"phase": "design"}},
    {"text": "query2", "filters": {"phase": "develop"}}
  ]
}
```

### Discovery Stage
```json
"discovery": {
  "siblings": true,              // Same file_path (0.9 weight)
  "genealogy": true,             // Same session_id from conversations (0.85 weight)
  "temporal": true,              // Semantic + chronological (0.85+ score)
  "temporal_direction": "after", // Optional: "after" | "before"
  "cross_phase": "design"        // Search in different phase (0.7+ score)
}
```

### Graph Stage
```json
"graph": {
  "algorithm": "authority",  // Reference counting (siblings + genealogy)
  "top": 5                   // Optional: Limit to top N by authority
}
```

### Output Stage
```json
"output": {
  "template": "genealogy"  // "genealogy" | "timeline"
}
```

**Without template:** Returns raw JSON

## Performance Expectations

**Single query + discovery:** ~100ms
**Multi-query (3 parallel):** ~60ms (3× faster than sequential)
**Full discovery (all ops):** ~150ms
**Search only (no discovery):** ~40ms

**Bottleneck:** Encoder inference (~30ms per query), not Qdrant

---

## What to Look For

###  Success Indicators
- Results returned < 200ms
- Sibling counts: 10-25 per document
- Temporal results: 5-20 related chunks
- Authority scores differ from semantic scores
- Templates render readable markdown

### L Failure Indicators
- Timeout or > 500ms
- Empty results for known queries
- Duplicate chunks in siblings
- Authority ranking identical to semantic
- Template rendering errors
