  ## Compositional Philosophy

  **FlexGraph = Flexible primitives, not prescribed patterns**

  ### The Building Blocks (Layer 1)
  - `siblings` - Related chunks from same document
  - `genealogy` - Origin conversation (via session_id)
  - `temporal` - Evolution over time
  - `cross_phase` - Related content in other phases

  ### Any Composition Valid (Layer 2)
  Agents compose primitives ANY way:

  **Example compositions discovered through usage:**

  1. Complete Story (30+ uses)
     ```json
     {"genealogy": true, "cross_phase": "design", "siblings": true}
     Returns: Conversation → Design → Implementation

  2. Evolution Timeline (20+ uses)
  {"temporal": true, "siblings": {"section_types": ["Patterns"]}}
  2. Returns: How thinking evolved
  3. Anti-Pattern Search (15+ uses)
  {"siblings": {"section_types": ["Failures"]}}
  3. Returns: What didn't work across docs
  4. Pattern Library (10+ uses)
  {"siblings": {"section_types": ["Patterns"], "order_by": "timestamp"}}
  4. Returns: Reusable learnings

  No composition is "wrong" - usage reveals what's valuable.

  Observable Usage → Preset Library (Layer 3)

  Not prescriptive. Discovery-driven.

  1. Agents use FlexGraph freely
  2. Track: Which compositions repeat?
  3. After 10-20 uses: Clear pattern emerges
  4. Capture: As slash command preset

  Example:
  Composition used 30 times:
  {"genealogy": true, "siblings": {"section_types": ["Decisions", "Failures"]}}

  Captured as: /explain-decision
  Usage: Reconstruct complete decision context

  Why This Matters

  Template-as-schema: Deterministic structure (v3 changelogs)
  FlexGraph: Flexible composition (any combination)
  Observable usage: Emergent patterns (proven by usage)

  Not "here's how to use it" but "discover what works."
