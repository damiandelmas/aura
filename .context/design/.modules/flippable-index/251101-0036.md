  ---
  Cross-Project Pattern Aggregation: ✅

  Works because:

  1. Same embedding space - All collections use same model (Nomic Embed v1.5)
  2. Vectors are comparable - Can merge/rank results across collections
  3. Language-agnostic - Pattern chunks designed for cross-project reuse

  Implementation:

  # Query all pattern collections
  pattern_collections = [
      "imem_project_a_pattern",
      "imem_project_b_pattern",
      "imem_project_c_pattern"
  ]

  # Search each, merge results
  all_results = []
  for collection in pattern_collections:
      results = search(collection, query, limit=20)
      all_results.extend(results)

  # Re-rank by score globally
  all_results.sort(key=lambda r: r['score'], reverse=True)
  return all_results[:limit]

  Semantic retrieval: Still works perfectly. Same semantic space.

  ---
  Cross-Project Impl Aggregation: ❌

  Doesn't work because:

  1. Project-specific context - "JWT auth" in FastAPI vs Go vs Rust
  2. Semantic pollution - Too many similar but irrelevant results
  3. Lost context - Need to know which project you're working in
  4. Ranking chaos - 1000s of chunks, hard to rank meaningfully

  Example failure:

  Query: "database connection pooling"

  Results from all impl collections:
  - Project A: PostgreSQL pool in Python
  - Project B: MySQL pool in Go
  - Project C: Redis pool in Node
  - Project D: MongoDB in Rust
  - ... 50 more results

  Problem: Which one is relevant? You need project context.

  ---
  Recommended Query Modes:

  # Mode 1: Current project impl (default)
  imem search "auth"
  → Query: imem_current_project_impl

  # Mode 2: Current project patterns
  imem search "auth" --patterns
  → Query: imem_current_project_pattern

  # Mode 3: Cross-project patterns (all projects)
  imem search "auth patterns" --cross-project
  → Query: [imem_*_pattern] (all pattern collections)

  # Mode 4: Specific project
  imem search "auth" --project project_b
  → Query: imem_project_b_impl

  ---
  Multi-Collection Search (Qdrant)

  Qdrant doesn't natively support multi-collection search in one query, so you need to:

  async def multi_collection_search(collections, query, limit=10):
      """Search multiple collections in parallel, merge results"""

      # Search all collections in parallel
      tasks = [
          asyncio.to_thread(search_collection, col, query, limit * 2)
          for col in collections
      ]
      results_by_collection = await asyncio.gather(*tasks)

      # Flatten and merge
      all_results = []
      for results in results_by_collection:
          all_results.extend(results)

      # Re-rank by score
      all_results.sort(key=lambda r: r['score'], reverse=True)

      # Return top-k globally
      return all_results[:limit]

  Performance: Fast. Parallel queries + merge is ~50-100ms.
