### Phase 6: Soft-Graph Object Model

**Goal:** Enable graph-like navigation via metadata queries

**Effort:** ~200 lines, 2 days

**Components:**

1. **Base Classes** (+80 lines)
   - `KnowledgeGraph` class (project initialization, search interface)
   - `Item` base class (payload wrapping, relationship properties)
   - `Pattern` subclass (source implementation link)
   - `Discussion` subclass (resulting changelog link)

2. **Lazy-Loaded Relationships** (+60 lines)
   - `Item.constraints` (filter: same file_path + section_type='Constraints')
   - `Item.pattern_layer` (naming: .md → .pattern.md)
   - `Item.origin_discussion` (filter: session_id match)
   - `Item.later_evolution` (filter: timestamp > self + semantic similarity)

3. **Quality Scoring** (+30 lines)
   - `Item.quality_score` property
   - Supersession detection (later_evolution check)
   - Constraint count penalty
   - Pattern existence boost

4. **Caching Layer** (+30 lines)
   - Cache relationship queries after first access
   - Session-scoped cache (cleared between searches)
   - Avoid redundant Qdrant queries

**Files:**
- `imem/src/imem/graph.py` (new)
- `imem/src/imem/__init__.py` (export KnowledgeGraph)

**Validation:**
```python
from imem import KnowledgeGraph

graph = KnowledgeGraph("~/my-project")

# Semantic search
decision = graph.search("JWT authentication")[0]

# Navigate relationships
assert len(decision.constraints) > 0
assert decision.pattern_layer is not None
assert decision.origin_discussion is not None

# Quality score
score = decision.quality_score
assert 0.0 <= score <= 1.5

# Chaining
constraint = decision.constraints[0]
conversation = constraint.origin_discussion
assert conversation.payload['source'] == 'conversation'
```

**Deliverables:**
- ✅ KnowledgeGraph API for semantic search
- ✅ Item object model with lazy relationships
- ✅ Dynamic navigation via metadata queries
- ✅ Quality scoring for authority ranking

### Phase 7: Multi-Modal Retrieval

**Goal:** Implement explain/trace/patterns retrieval modes

**Effort:** ~150 lines, 2 days

**Components:**

1. **EXPLAIN Mode** (+50 lines)
   - Find primary match (semantic search)
   - Get related sections (same file_path)
   - Get pattern layer (.pattern.md variant)
   - Get origin conversation (session_id match)
   - Format complete context bundle

2. **TRACE Mode** (+60 lines)
   - Find decision (semantic + phase=develop)
   - Get conversation (session_id match, earliest timestamp)
   - Get design docs (session_id + phase=design)
   - Get designate plan (session_id + phase=designate)
   - Get later refinements (timestamp > decision + semantic)
   - Return chronological timeline

3. **PATTERNS Mode** (+40 lines)
   - Find implementation (semantic + layer=implementation)
   - Find pattern variant (.pattern.md naming)
   - Find similar patterns across projects (cross-collection search)
   - Return comparison bundle

**Files:**
- `imem/src/imem/retrieval_modes.py` (new)
- `imem/src/imem/cli.py` (add subcommands)

**Validation:**

```bash
# Returns: Conversation → Design → Plan → Implementation → Refinements
```

### Phase 9: Supersession Detection

**Goal:** Automatically detect when decisions are replaced

**Effort:** ~100 lines, 1-2 days

**Components:**

1. **Detection Algorithm** (+40 lines)
   - On new decision ingestion, search for similar existing decisions
   - Threshold: 0.85 semantic similarity
   - If found, mark older decision with `superseded_by` metadata

2. **Quality Impact** (+20 lines)
   - Update quality_score to heavily penalize superseded items
   - Rank superseded items lower in search results
   - Optionally hide superseded items (configurable)

3. **User Confirmation** (+40 lines)
   - Prompt user: "Does this replace [old decision]?"
   - Store supersession relationship if confirmed
   - Skip if user rejects

**Files:**
- `imem/src/imem/supersession.py` (new)
- `imem/src/imem/ingest.py` (integrate detection)

**Validation:**
```bash
# Index initial decision
imem init  # Indexes "Use JWT Auth" decision

# Index replacement decision
# Add new changelog: "Migrate to OAuth2"
imem init
# Prompt: "Does this replace 'Use JWT Auth'? [y/N]"
# If yes: Marks old decision as superseded

# Search shows newer decision ranked higher
imem search "authentication"
# Result 1: OAuth2 (current)
# Result 2: JWT (superseded, lower quality score)
```

**Deliverables:**
- ✅ Automatic supersession detection
- ✅ User confirmation workflow
- ✅ Quality score impact
- ✅ Optional hiding of superseded items


## Risk Assessment

### High Risk

**Template compliance dependency:**
- System relies on developers following templates
- Without validation (Phase 8), metadata reliability breaks
- **Mitigation:** Prioritize Phase 8 immediately after Phase 5A+B

**LlamaIndex metadata parsing:**
- header_level not populated in v0.14.5
- Need to parse manually via regex
- **Mitigation:** Tested workaround exists (`ingest.py:695-697`)

### Medium Risk

**Soft-graph performance:**
- Metadata queries slower than indexed edges
- Multiple relationship traversals could be slow
- **Mitigation:** Session-scoped caching (Phase 6)

**Cross-collection search:**
- Qdrant doesn't natively support multi-collection queries
- Need custom aggregation logic
- **Mitigation:** Phase 11 marked low priority

### Low Risk

**Supersession false positives:**
- Semantic similarity threshold may trigger incorrectly
- User confirmation workflow mitigates
- **Mitigation:** Configurable threshold (default 0.85)
