# Template-as-Schema: The Foundational Moat

**Status:** Non-negotiable architectural foundation

---

## The Geometric Shift

**Traditional RAG:**
```
Content → Extract → Maybe metadata (probabilistic)
```

**IMEM:**
```
Template → Enforce → Guaranteed metadata (deterministic)
```

**The moat:** Mandatory structure, not better extraction.

---

## Deterministic vs Probabilistic

**Traditional systems:**
- LLM extracts metadata post-hoc
- Field presence = probability
- Queries return partial results

**IMEM:**
- Template defines schema upfront
- Field presence = guarantee
- Queries return complete results

**Property:** Deterministic filtering on vector search.

---

## The Foundation

Everything builds on guaranteed metadata:
- Graph relationships (metadata queries)
- Supersession detection (field presence)
- Authority scoring (complete context)
- Cross-project patterns (consistent structure)

**Break template enforcement → architecture collapses.**

---

## The Moat

**Unique capability:** Deterministic queries on vectors.

No other system guarantees field presence. All others = post-hoc extraction (probabilistic, lossy).

**This is the differentiator.**
