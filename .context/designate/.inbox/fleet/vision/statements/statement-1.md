# System Shape: Universal Knowledge Foundation

## Hypothesis

Structured preservation at write-time enables cheaper, higher-quality knowledge graph construction than post-hoc extraction.

## Core Dataflow

### Foundation Layer (Layer 1)

- CORE classification (1 call per chunk)
- Semantic types + domain types + coordinates
- Metadata creates programmatic queryability
- Result: Indexed, typed chunks

↓

### Tools Layer (Layer 2)

- `compose()` queries foundation
- Semantic + programmatic filters
- Metadata queries (no LLM calls)
- Result: Typed result sets

↓

### Intelligence Layer (Layer 3)

- AI agents query structured chunks
- Extract entities/relationships from typed input
- Hypothesis: 5-10 calls for entire KG (vs 50 per doc)
- Guided extraction from structure

---

## Architectural Points

### Three-Tier Foundation

- **Tier 0**: All intellectual capital (equal treatment)
- **Tier 1**: Objective registries ABOUT sources
- **Tier 2**: Subjective gateways - authority at serve time

### MIND Intelligence (Orthogonal)

1. **Schema Evolution**: Observe → Cluster → Emerge types
2. **Entity Resolution**: Variants → Canonical → Query expansion
3. **Introspection**: Expose capabilities programmatically
4. **Runtime Graphs**: Metadata → Ephemeral edges → Algorithms
5. **Temporal Cortex**: Git diffs → Truth validation → Drift detection

---

## Key Principles

- Economics hypothesis: 1 call foundation vs 50 calls extraction
- Composition approach: Ephemeral graphs, query-scoped
- Self-describing: introspect → capabilities
- Cross-domain: CORE patterns portable
- Validation: Code is ground truth

---

## System Flow

```text
Raw docs → CORE (1 call) → Programmatic foundation
        ↓
    AI agents with tools query typed chunks
        ↓
    Construct KG from structured input
        ↓
    50x cheaper, higher quality
```

## The Architectural Approach

Structure preservation at creation, not extraction from chaos.

Hypothesis: Typed foundations enable AI to compose graphs more efficiently from structured input.
