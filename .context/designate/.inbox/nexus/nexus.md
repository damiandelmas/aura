## Essential Geometry: NEXUS

---

### Core Primitives

**1. ALL SOURCES = INTELLECTUAL CAPITAL (Tier 0)**
- Documents, URLs, folders, repos, papers, logs
- No hierarchy at this layer
- Heterogeneous, as they exist
- Equal potential value

**2. THREE-TIER GATEWAY**
```
Tier 0: Thing itself
Tier 1: Objective facts ABOUT thing
Tier 2: Subjective relationship TO thing
```

*Note: Tier 1 may emerge from queries. When a project accesses a source, Tier 1 entry is created (if not exists) and Tier 2 entry captures usage metadata.*

**3. AUTHORITY EMERGES AT SERVE TIME**
- Not intrinsic to source
- Determined by context + usage
- Same source, different authority per project

**4. HARMONIOUS LAYERS USE NEXUS**
- IMEM (institutional memory with MIND capabilities)
- TRACE (session genealogy)
- PROCESS (four-phase workflow)
- All exploit NEXUS infrastructure, provide specialized capabilities

**5. GIT = TRUTH SPINE**
- Commits provide temporal ordering
- Diffs validate claims
- Implementation grounds documentation

---

```
  HARMONIOUS LAYERS
         │
    ┌────┴────┐
    │         │
   IMEM     TRACE
  (MIND)  (genealogy)
```

---

### Essential Properties

**Property 1: Separation of Concerns**
- NEXUS = Universal wrapper (tiers, visibility, authority)
- Harmonious layers = Specialized capabilities (IMEM/MIND, TRACE, PROCESS)

**Property 2: Contextual Authority**
```
Develop log in origin project = Ground truth
Same log in other project = Reference example
```
Authority from relationship, not declaration.

**Property 3: Metadata Visibility**
Source enters system → Tier 1 registry created → Queryable
No content processing required for visibility.

**Property 4: Selective Depth**
```
NEXUS: All sources, metadata, visibility (universal)
IMEM: Selected sources, full indexing, MIND intelligence (deep)
TRACE: Session genealogy, temporal linkage (specialized)
```
Visibility everywhere. Depth selectively.

**Property 5: Validation Through Reality**
```
Document predicts → Code implements → Git records → Validation score emerges
```
Truth discovered, not declared.

---

### The Three Essential Operations

**1. REGISTER**
```
Source encountered → Tier 1 entry created → Metadata indexed
Result: Visible, queryable
```

**2. QUALIFY**
```
Project accesses → Tier 2 wrapper created/updated → Usage accumulates
Result: Authority emerges
```

**3. VALIDATE**
```
Documentation → Implementation → Git diff → Divergence computed
Result: Truth grounded
```

---

### Key Insights

**Insight 1: No Precomputation**
- Schemas emerge from observation
- Graphs materialize on-demand
- Entities resolve during query
- Authority accumulates through usage

**Insight 2: NEXUS Independent of Capabilities**
```
NEXUS wraps all interactions regardless of:
  - Retrieval method (IMEM, WebFetch, Read, Git)
  - Intelligence layer (MIND for IMEM)
  - Methodology (PROCESS, research, agile)
```
Harmonious layers are pluggable.

**Insight 3: Multi-Project Scale**
```
Single repo: Four-phase lifecycle
Fleet of repos: NEXUS metadata network
```
Same principles, different scopes.

**Insight 4: Two Networks**
```
KNOWLEDGE (metadata): What exists, when accessed, by whom
CONTENT (indexed): Semantic meaning, deep search
```
Superset → Subset relationship.
