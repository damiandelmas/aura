## Essential Geometry

### Core Primitives

**1. PROJECT = GIT REPOSITORY**
- Boundary: `.git/` directory
- Ground truth: Commit history
- Isolation: Per-project knowledge

**2. KNOWLEDGE TRANSFORMS THROUGH PHASES**
```
design → designate → develop → document
(fluid)   (crystal)   (ground)  (stable)
```

**3. EACH PHASE HAS DISTINCT ABSTRACTION LAYER**
- Design: Questions, alternatives, decisions
- Designate: Clear plans, ready to execute
- Develop: What was built, what happened
- Document: How it works, how to use

**4. MANAGE (Universal Wrapper)**
- Implemented via NEXUS (.nexus/ folder, metadata network)
- Tier 0→1→2 wrapping (visibility + context)
- Authority emergence at serve time
- Orchestrates all interactions (knowledge, tools, agents)

**5. CONTEXT = MULTI-SOURCE SYNTHESIS**
- Git commits (what changed)
- Phase documents (why/how/what)
- Conversations (unrealized ideas)
- → Chronological narrative

### Essential Properties

**Property 1: Phase Separation**
- Design ≠ Designate (exploration ≠ plan)
- Designate ≠ Develop (plan ≠ implementation)
- Develop ≠ Document (ground truth ≠ reference)

**Property 2: Knowledge Flow**
```
Fluid exploration
  ↓ crystallizes
Clear specification
  ↓ materializes
Ground truth
  ↓ stabilizes
Stable reference
```

**Property 3: Context Emergence**
Understanding emerges from synthesis, not declarations.
- Single source = partial view
- Multi-source + phases = complete lineage

**Property 4: Retrieval Dimensionality**
Knowledge is retrievable across:
- Phase (design/designate/develop/document)
- Time (chronological)
- Genealogy (session linkage)
- Type (24 document patterns)

### The Three Essential Operations

**1. CREATE**
```
Human + AI → Produces artifact → Enters via entry mechanisms
```

**2. MANAGE**
```
Source enters → Tier 1 registry → Tier 2 qualification → Visible + contextualized
Implementation: NEXUS handles tier wrapping
```

**3. USE**
```
Agent action → Retrieval/tool/orchestration → MANAGE qualifies → Serve
```

**4. SYNTHESIZE**
```
Query → Multi-phase search → Chronological assembly → Genealogical context
```

### Key Insights

**Insight 1: Designate Fills The Gap**
```
Without: design → ??? → develop
With:    design → designate → develop
         (rough)  (clear)     (ground)
```

**Insight 2: Git Anchors Everything**
- Repository = project boundary
- Commits = objective timeline
- Branches = feature isolation
- History = validation oracle

**Insight 3: Phases = Abstraction Layers**
Not temporal stages - different ways of thinking about same thing:
- Design: What should we do?
- Designate: Here's the clear plan.
- Develop: Here's what happened.
- Document: How it works.

**Insight 4: Lineage = Onboarding**
Complete genealogy enables:
- Any agent joins anytime
- Full context reconstruction
- Decision rationale visible
- Alternatives understood

**Insight 5: Harmonious Layers**
- PROCESS (four-phase) produces artifacts
- IMEM (with MIND) provides deep retrieval
- TRACE provides genealogy
- All use MANAGE (via NEXUS), all separate from core
