## Core Primitives

**1. PROJECT = GIT REPOSITORY**
- Boundary: `.git/` directory
- Truth: Commit history validates claims
- Universal: Any codebase, any language

**2. THREE SOURCES OF KNOWLEDGE**
```
Markdown files  (intent + decisions)
Conversations   (lineage + context)
Git history     (ground truth)
```

**3. CANONICAL SCHEMA (Four-Phase)**
```
design → designate → develop → document
(explore)  (specify)   (ground)   (stable)

Universal target for ALL workflows
```

**4. CHUNKS = ATOMIC UNIT**
```
Document metadata: type, subtype, phase
Chunk metadata: section_type, inherited context
```

**5. RESOLUTION NOT VALIDATION**
```
Any markdown → Observe → Infer → Map to canonical
Parser discovers intent, doesn't enforce compliance
```
