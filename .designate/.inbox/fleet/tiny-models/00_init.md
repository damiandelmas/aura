## Core Insights

**Type Inference Beyond Code**
- Hindley-Milner for documents: observe patterns, infer structure
- Universal foundations (CORE) resolve to domain-specific types contextually

**Six Dimensions as Universal Algebra**
- Interrogative, valence, abstraction, epistemic, temporal, structural
- Domain types emerge as coordinates in CORE-space
- Resolution happens at serve time, not index time

**Authority Is Emergent**
- Same source, different qualification per project
- Context determines meaning, not intrinsic properties

**Foundation Models Bootstrap, Tiny Models Scale**
- Foundation: Semantic structuring (one-time, ~$0.01/chunk)
- Tiny: Classification, polish, specialized tasks (free after ~$200 training)
- Agents: Intelligent composition on structured foundation

**The Moment**
- nanochat: $100-1000 trains ChatGPT-like models
- TRM: 7M params + recursion competes with massive models
- Economics shifted: Train custom > Pay APIs

## Architectural Sketch

**Three-Tier + BRAIN**
```
Tier 0: Raw sources
Tier 1: Normalized registry
Tier 2: Per-project qualification
BRAIN: Orthogonal intelligence (schema evolution, graphs, temporal validation)
```

**CORE Flow**
```
Index: 1 call → 6D coords
Store: Rich metadata foundation
Query: CORE + context → domain type
Serve: Project attention qualifies
```

**Layered Intelligence**
```
Tiny models: Add structure
Metadata: Enable queries
AI agents: Compose structures
Runtime: Resolve contextually
```
