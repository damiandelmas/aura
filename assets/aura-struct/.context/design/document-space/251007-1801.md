  The Shape: Institutional Memory as Living Documentation

  Core Insight from AURA Structure

  The system manages three temporal states of knowledge:

  .design/     → What we're EXPLORING (R&D, might fail)
  .develop/    → What we're DOING (ground truth, emergent patterns)
  .document/   → What we KNOW (stable, integrated, authoritative)

  Plus Three Content Layers (within .document/):

  schemas/         → WHAT IS (immutable ground truth)
  business-logic/  → WHAT DOES (workflows, operations)
  architecture/    → HOW & WHY (structure, design decisions)

  ---
  The Architecture: Three Intelligences

  1. TRACE - Conversation Archaeology

  Purpose: Extract context from development reality

  Shape:
  - Reads Claude Code JSONL (the actual work that happened)
  - Provides context for agents ("what did we discuss?")
  - Enables conversation-aware documentation

  Flow:
  Development Session (Reality)
      ↓
  JSONL files (Ground Truth)
      ↓
  TRACE extracts context
      ↓
  Feeds to Agent System

  ---
  2. PULSE - Document Transformation

  Purpose: Transform temporal knowledge into stable knowledge

  Shape (THIS IS KEY):
  Input:  .develop/.changes/YYMMDD-HHMM_session.md
             ↓
  PULSE reads changelog
             ↓
  Decides: Which layer needs updating?
             ↓
  Output: .document/schemas/X.md         (if ground truth changed)
          .document/business-logic/Y.md  (if workflow discovered)
          .document/architecture/Z.md    (if design evolved)

  PULSE is intelligent transformation:
  - Not just "move file from changes to snapshot"
  - Understands the three-layer taxonomy
  - Routes changes to correct document layer
  - Integrates new knowledge into existing structure

  ---
  3. IMEM - Search & Discovery

  Purpose: Make all knowledge discoverable

  Shape:
  Indexes:
    .design/     (exploratory knowledge)
    .develop/    (temporal knowledge)
    .document/   (stable knowledge)

  Enables:
    "Find all decisions about agent management"
    "What patterns emerged in Sept?"
    "What's our current auth schema?"

  The Agent System Shape (Conceptual)

  Agent System as Orchestrator

  ┌─────────────────────────────────────┐
  │     AGENT ORCHESTRATOR              │
  │  (fires when needed, coordinates)   │
  └─────────────────────────────────────┘
           ↓                    ↓
      ┌────────┐          ┌────────┐
      │ TRACE  │          │ PULSE  │
      │ (read) │          │ (write)│
      └────────┘          └────────┘
           ↓                    ↓
      Context          Transformed Docs

  Agent Types (from your docs):
  1. ChangelogAgent - Creates .develop/.changes/ from TRACE context
  2. PulseAgent - Reads changelog, updates .document/ layers (schema/logic/arch)
  3. PruneAgent - Updates metadata on old .develop/.changes/

  Key Insight: Agents use TRACE and PULSE, don't replace them.

  The Refined Concept (No Implementation)

  Core Components:

  1. TRACE - Read-only conversation archaeology
  2. PULSE - Intelligent document transformation (understands 3-layer taxonomy)
  3. IMEM - Search & discovery across all layers
  4. Agent Orchestrator - Coordinates the pipeline (uses TRACE, fires PULSE)

  Data Flow:

  Reality (.develop/.changes/)
      ↓
  TRACE reads it
      ↓
  Agent System processes it
      ↓
  PULSE transforms to .document/
      ↓
  IMEM makes it searchable

  Folder Structure:

  project/
  ├── .design/
  │   ├── .changes/      # R&D explorations
  │   └── .modules/      # Experimental approaches
  ├── .develop/
  │   ├── .changes/      # What happened (ground truth)
  │   └── .modules/      # Emergent patterns (proven, not yet integrated)
  └── .document/         # Current stable state
      ├── [simple docs initially]
      └── [extract schemas/logic/arch when pain emerges]
