● Overall Workflow

Four-phase transformation: Fluid exploration crystallizes into structured plan, materializes through
execution, solidifies into persistent reference.

Separation principle: Each phase operates at distinct abstraction layer with unique constraint profile.
Upstream quality determines downstream effort. Early phase completeness enables later phase speed.

Flow: Exploration space → specification space → implementation space → reference space.

---
Phase Essences

Phase 1: Exploration Space

Navigates possibility terrain. Multiple paths evaluated. Decisions emerge through investigation and
comparison. Operates at conceptual/geometric layer - shapes, relationships, boundaries. Resists premature
descent into specifics. Primary mode: expansion then contraction. Output: decision topology with rationale
map.

Constraint profile: Stay abstract. Qualify uncertainty. Resist implementation pull. Research before creating.

Separation: Thinking domain. Non-executable. Conceptual architecture only.

---
Phase 2: Specification Space

Transforms exploration decisions into executable plans. Crystallization from conceptual to actionable. The
bridge: abstract understanding → concrete instruction. Refines rough structure into precise guidance. Creates
artifacts others can execute without interpretation gaps.

Constraint profile: Clear articulation. Remove ambiguity. Define boundaries precisely.

Separation: Planning domain. Blueprint layer. Not execution, not exploration - the intermediary solid.

---
Phase 3: Execution Space

Materializes plan into reality with minimal deviation. Preservation fidelity over creative enhancement.
One-shot achievable when specification space was complete. Documents what-happened alongside what-was-built.
Captures divergence from plan when necessary.

Constraint profile: Faithful implementation. Document deviations. No speculative addition.

Separation: Doing domain. Creates artifacts, not designs them. Executes, not explores.

---
Phase 4: Reference Space

Persistent orientation layer. Static snapshots that can drift. Updated deliberately, not continuously.
Provides shape understanding for future context. Plain conceptual clarity over technical precision. Minimal
complete overviews.

Constraint profile: Stable reference. Simple language. Onboarding orientation.

Separation: Knowledge domain. Not work artifacts - maps of work artifacts.

---
Delineation

Phase 1 ≠ Phase 2: Exploration stays geometric/conceptual. Specification crystallizes into actionable
structure. Thinking ≠ Planning.

Phase 2 ≠ Phase 3: Specification creates instructions. Execution follows instructions. Blueprint ≠ Building.

Phase 3 ≠ Phase 4: Execution produces work artifacts + logs. Documentation maps the landscape. Ground truth ≠
Reference map.

Quality cascade: Phase 1 quality determines Phase 3 speed. Complete specification enables one-shot execution.
Incomplete exploration requires iteration.
