## Click Groups: Best Practice Analysis

### When Click Groups ARE Good Practice

**Multi-verb CLIs** (git, docker, kubectl style):
```bash
git commit -m "..."
git push origin main
docker build -t image .
docker run image
```

**Benefits:**
- Clear separation of concerns
- Each subcommand has focused responsibility
- Easy to extend (add new subcommands)
- Better help organization
- Command-specific options don't pollute global namespace

### When Single Commands Work Better

**Query/filter CLIs** (grep, find, jq style):
```bash
grep -r "pattern" --include="*.py"
find . -name "*.md" -type f
jq '.data[] | select(.active)' 
```

**Benefits:**
- Simpler mental model (one thing, many filters)
- Faster typing (no verb needed)
- Natural for "search and display" tools
