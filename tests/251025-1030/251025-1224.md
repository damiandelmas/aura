    - **Pattern**: Use JSONB column with GIN index instead of enum columns for provider-specific attributes
    - **When**: Schema needs to support multiple providers with different categorization schemes
    - **Approach**: Store provider-specific data as `metadata JSONB`, filter via `metadata @> '{"key": "value"}'`, index with `CREATE INDEX USING GIN(metadata)`
